Projeto acadêmico e laboratorial focado em **Segurança em Aplicações Web e Dispositivos Móveis**, desenvolvido a partir da disciplina universitária de **Segurança em Ambiente Web e Mobile**.

O objetivo deste repositório é **documentar conceitos, vulnerabilidades, práticas de defesa e experimentos práticos** relacionados à segurança da informação.

Este projeto combina:

* 📚 Base acadêmica
* 🧪 Laboratórios práticos
* 🛡 Frameworks de segurança reconhecidos
* 🇧🇷 Regulamentação brasileira de proteção de dados

---

# 🎯 Objetivos do Projeto

Este laboratório foi criado para:

* Estudar **fundamentos de segurança web**
* Compreender **vulnerabilidades comuns**
* Demonstrar **ataques controlados em ambiente de laboratório**
* Aplicar **técnicas de mitigação**
* Produzir **documentação técnica profissional**

# 🧠 Frameworks e Referências de Segurança

Este projeto também utiliza conceitos presentes em frameworks reconhecidos internacionalmente.

## MITRE ATT&CK

Base de conhecimento global sobre **táticas e técnicas utilizadas por adversários** em ambientes computacionais.

Utilizado para compreender:

* técnicas de ataque
* comportamento de ameaças
* modelagem de adversários

---

## NIST Cybersecurity Framework

Modelo de gestão de segurança baseado em cinco funções principais:

1. Identify
2. Protect
3. Detect
4. Respond
5. Recover

Este framework é amplamente utilizado para **governança de segurança da informação**.

---

## OWASP

Referência mundial em **segurança de aplicações web**.

Este projeto considera vulnerabilidades conhecidas como:

* SQL Injection
* Broken Authentication
* Session Management Issues
* Input Validation Failures

---

# 🇧🇷 Regulamentação Brasileira

O projeto também considera princípios da legislação brasileira relacionada à segurança da informação.

## LGPD – Lei Geral de Proteção de Dados

A LGPD regula o **tratamento de dados pessoais no Brasil** e estabelece princípios fundamentais como:

* proteção da privacidade
* segurança da informação
* minimização de dados
* transparência no tratamento de dados

Este projeto considera boas práticas relacionadas à proteção de dados.

# 🧪 Laboratórios Práticos

Este repositório inclui experimentos práticos de segurança.

## SQL Injection Lab

Demonstra como falhas na validação de entrada podem permitir manipulação de consultas SQL.

Conteúdos:

* exemplo de vulnerabilidade
* exploração controlada
* mitigação com prepared statements

---

## Authentication Bypass

Demonstra falhas em mecanismos de autenticação.

Conteúdos:

* bypass de login
* falhas de validação
* boas práticas de autenticação segura

---

## Session Management Lab

Explora riscos associados ao gerenciamento de sessões.

Conteúdos:

* session hijacking
* cookies inseguros
* mitigação com HttpOnly e Secure
